Before committing, do the following:

1. Run `git diff --staged` to see what's changed
2. Run the project's lint and tests (e.g. `npm run lint && npm test`)
3. If anything fails, report it and stop
4. If everything passes, generate an appropriate commit message following conventional commits format (feat/fix/chore/refactor etc.)
5. Run `git commit -m "<generated message>"`
6. Run `git push`
